There are four files associated with this project (not including this one):

  • Udacity_AZDIAS_Subset.csv: Demographics data for the general population of Germany; 891211 persons (rows) x 85 features (columns).

# Simple function to return NAN if this is in the nan feature info set
def get_na(i, j):
    if i in j:
        return np.nan
    else:
        return i

# Set unknowns to NaN 
for c in azdias_c.columns:
    try:
        feature = feat_info_c[feat_info_c.attribute==c].missing_or_unknown.values[0]

        azdias_c[c] = azdias_c[c].apply(lambda x: get_na(x, j))

        print("Failed to process column: {0} with values: {1}".format(c, sub))

def clean_data(df):
    """
    Perform feature trimming, re-encoding, and engineering for demographics
    data
    
    INPUT: Demographics DataFrame
    OUTPUT: Trimmed and cleaned demographics DataFrame
    """
    feat_info_copy = feat_info.copy()
    # Put in code here to execute all main cleaning steps:
    # convert missing value codes into NaNs, ...
    for c in df.columns:
        try:
            feature = feat_info_copy[feat_info_copy.attribute==c].missing_or_unknown.values[0]
            sub = re.sub(r'\bX\b', '\"X\"', feature) # found these Xs in steps above, wasn;t enough to warrant general solution
            sub = re.sub(r'\bXX\b', '\"XX\"', sub) # found these XXs in steps above, wasn;t enough to warrant general solution
            j = json.loads(sub)
            df[c] = df[c].apply(lambda x: get_na(x, j))
        except:
            print("Failed to process column: {0} with values: {1}".format(c, sub))
    
    # remove selected columns and rows, ...
    nans = []
    df_length = len(df[c])
    for c in df.columns:
        l = len(df[df[c].isna()][c])
        #if l > 0:
        nans.append([c, l, df_length, l/df_length])
    df_nan = pd.DataFrame(nans, columns=["attribute", "nans", "total", "percentage"])
    df_nan["fPercentage"] = df_nan.percentage.map('{:.2%}'.format)
    df_nan = df_nan.sort_values(by="percentage", ascending=False)
    anomolies = list(df_nan.head(6).attribute)
    df.drop(anomolies, axis=1, inplace=True)
    for c in anomolies:
        idx = feat_info_copy[feat_info_copy.attribute == c].index
        feat_info_copy.drop(index=idx, axis=0, inplace=True)    
    df["nbr_na_cols"] = df.isna().sum(axis=1)
    
    # select, re-encode, and engineer column values.
    feat_cats = feat_info_copy[feat_info_copy.type=='categorical'][['attribute', 'type']]
    vals = []
    for c in feat_cats.attribute:
        u = df[df[c].isna()==False][c].unique()
        vals.append([c, u, len(u), u.dtype])
    df_uniques = pd.DataFrame(vals, columns=['attribute', 'uniques', 'unique_count', 'data_type'])
    df_multi = df_uniques[~((df_uniques.unique_count==2)&(df_uniques.data_type!='object'))]
    df = pd.get_dummies(df,columns=df_multi.attribute)

    df['PRAEGENDE_JUGENDJAHRE_DEC'] = df.PRAEGENDE_JUGENDJAHRE.apply(lambda x: get_decade(x))
    df['PRAEGENDE_JUGENDJAHRE_MOV'] = df.PRAEGENDE_JUGENDJAHRE.apply(lambda x: get_movement(x))
    
    df.CAMEO_INTL_2015 = df.CAMEO_INTL_2015.astype(float) #convert to float for comparison

    df['CAMEO_INTL_2015_WEALTH'] = df.CAMEO_INTL_2015.apply(lambda x: get_wealth(x))
    
    # Return the cleaned dataframe.

    df.drop('nbr_na_cols', axis=1, inplace=True)
        
    del df_nan
    del df_length
    del feat_cats
    del df_uniques
    del df_multi
    return df

Step 2.3: Interpret Principal Components¶

Now that we have our transformed principal components, it's a nice idea to check out the weight of each variable on the first few components to see if they can be interpreted in some fashion.

As a reminder, each principal component is a unit vector that points in the direction of highest variance (after accounting for the variance captured by earlier principal components). The further a weight is from zero, the more the principal component is in the direction of the corresponding feature. If two features have large weights of the same sign (both positive or both negative), then increases in one tend expect to be associated with increases in the other. To contrast, features with different signs can be expected to show a negative correlation: increases in one variable should result in a decrease in the other.

  • To investigate the features, you should map each weight to their corresponding feature name, then sort the features according to weight. The most interesting features for each principal component, then, will be those at the beginning and end of the sorted list. Use the data dictionary document to help you understand these most prominent features, their relationships, and what a positive or negative value on the principal component might indicate.
